  • czo项目规范

    # 1.每个vue文件中的css和template中要进行单独模块的注释，纯js小于200行，不包括伪测试数据字段但不能太长不能超过（25行）//拆组件引用等，肯定能满足

    # 定义变量应从template从上到下进行定义，变量命名应在后面添加注释以说明

    # 2.特殊函数要进行函数的注释，常用的没有必要，但是methods中的方法要根据template模板进行从上到下的顺序命名并编辑

    # 3.能写在后台的逻辑不要放到前台实现（可根据具体需求进行前端或者后端进行存储）

    # 5.无用代码如console.log('xxxx')在打包后一定要删除掉，一些没有用到的代码块应进行注释(并添加备注说明)

    # 6.样式代码根据当前组件的页面进行编写，如果出现样式代码过多的情况应使用scss样式进行重构或编写(或者使用组件的方式进行引入）

    # 7.样式代码引入应尽量引入全局公共样式。

    • 1设置为true/男
